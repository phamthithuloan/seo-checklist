  Trên UI:
  1. Vào tab Review bài viết → nhập keyword + content → bấm Phân tích (hoặc mở 1

  pass/warn/fail, và toàn bộ chi tiết của từng rule (kèm các điểm chưa đạt nếu
  có)

  Nếu cần PDF: mở file .html bằng Chrome/Safari → File → Print → Save as PDF.

⏺ ---
  Hướng dẫn deploy (từng bước)

  Bước 0 — Push code lên GitHub

  Chạy trong terminal:
  ! git push origin main
  Repo: https://github.com/phamthithuloan/seo-checklist

  Bước 1 — Deploy Backend lên Railway

  1. Vào https://railway.app → đăng nhập GitHub → New Project → Deploy from 
  GitHub repo → chọn seo-checklist.
  2. Settings → Root Directory: đặt backend
  3. Railway tự nhận Dockerfile và build.
  4. + Add Service → Database → PostgreSQL (cùng project).
  5. Vào service backend → Variables → add:

  ┌─────────────────────────────────┬────────────────────────────────────────┐
  │               Key               │                 Value                  │
  ├─────────────────────────────────┼────────────────────────────────────────┤
  │ DATABASE_URL                    │ ${{Postgres.DATABASE_URL}} (reference, │
  │                                 │  gõ đúng cú pháp)                      │
  ├─────────────────────────────────┼────────────────────────────────────────┤
  │ JWT_SECRET                      │ output của openssl rand -hex 32        │
  ├─────────────────────────────────┼────────────────────────────────────────┤
  │ JWT_ACCESS_TOKEN_EXPIRE_MINUTES │ 10080                                  │
  ├─────────────────────────────────┼────────────────────────────────────────┤
  │ CORS_ORIGINS                    │ tạm *, sau khi có Vercel URL thay bằng │
  │                                 │  URL đó                                │
  ├─────────────────────────────────┼────────────────────────────────────────┤
  │ ANTHROPIC_API_KEY               │ (optional) key Anthropic nếu muốn bật  │
  │                                 │ AI proofread                           │
  ├─────────────────────────────────┼────────────────────────────────────────┤
  │ ENVIRONMENT                     │ production                             │
  └─────────────────────────────────┴────────────────────────────────────────┘

  6. Settings → Networking → Generate Domain → copy URL kiểu
  https://seo-checklist-backend-production.up.railway.app.
  7. Test: mở <URL>/health → phải trả {"status":"ok"}.

  Bước 2 — Deploy Frontend lên Vercel

  1. Vào https://vercel.com → Add New → Project → import seo-checklist.
  2. Root Directory: frontend. Framework Preset tự nhận Next.js.
  3. Environment Variables:

  ┌─────────────────────┬────────────────────────────────────────────┐
  │         Key         │                   Value                    │
  ├─────────────────────┼────────────────────────────────────────────┤
  │ NEXT_PUBLIC_API_URL │ URL Railway ở bước 1 (không có dấu / cuối) │
  └─────────────────────┴────────────────────────────────────────────┘

  4. Deploy. Sau ~2 phút có URL kiểu https://seo-checklist-xxx.vercel.app.

  Bước 3 — Đóng vòng CORS

  1. Quay lại Railway → backend service → Variables → sửa CORS_ORIGINS thành URL
   Vercel (ví dụ https://seo-checklist-xxx.vercel.app).
  2. Railway tự redeploy.

  Bước 4 — Smoke test production

  - Vào URL Vercel → register account mới → login → analyze 1 bài → kiểm tra tab
   Lịch sử → Export Markdown.
  - Nếu fail: mở DevTools → Network → xem request có gọi đúng Railway URL không,
   và CORS header trả về có chứa domain Vercel không.

  Bước 5 — Custom domain (optional)

  - Vercel: Project → Domains → add mindgate.yourdomain.com. Trỏ CNAME theo
  hướng dẫn.
  - Railway: Settings → Networking → Custom Domain → add
  api.mindgate.yourdomain.com. Nhớ thêm domain mới vào CORS_ORIGINS.

⏺ Q1: Tắt VS Code → có tự tắt local dev không?

  Tuỳ vào cách bạn start. Kiểm tra ps thấy:

  uvicorn PID 5964   PPID 1  ← orphan, độc lập VS Code
  next dev PID 10278 PPID 1  ← orphan, độc lập VS Code

  PPID 1 = parent process là init/launchd → không thuộc về VS Code. Tức 2
  process này hiện đang chạy nền độc lập, kể cả đóng VS Code chúng vẫn chạy. Có
  thể do lệnh được start qua terminal khác hoặc qua Claude Code's Bash từ
  session trước.

  Nguyên tắc chung:
  - Start trong VS Code integrated terminal → tắt VS Code thì tắt
  - Start trong Terminal.app / iTerm → không tắt cho đến khi đóng terminal đó
  - Start qua nohup / & / Claude Code Bash → orphan, chạy mãi đến khi reboot
  hoặc kill thủ công

  Dọn 2 process zombie hiện tại (recommend chạy ngay):
  ! lsof -ti:8000 -ti:3000 | xargs kill -9
